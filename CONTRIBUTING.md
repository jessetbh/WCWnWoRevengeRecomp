# Contributing

Thanks for your interest in improving WCW/nWo Revenge: Recompiled!

## Ground rules

- **Never commit ROM data or extracted game assets.** PRs containing game data will be
  closed. CI builds fetch the ROM from a private repository; fork PRs build without it.
- This project is licensed under GPLv3 (see `COPYING`); contributions are accepted
  under the same license.
- Start with `BUILDING.md` (how to build). For background on the techniques and
  invariants this port is built on, the sister project's development history is the
  best reference:
  [WCW vs. nWo World Tour: Recompiled](https://github.com/jessetbh/WCWvsNWOWorldTourRecomp)
  (`docs/devlog.md` there) — the two ports share their runtime stack, and nearly every
  design decision here was proven there first.

## Practical notes

- The runtime libraries (N64ModernRuntime, RecompFrontend, RT64) carry project-local
  fixes tagged `[wcw fix]` on the `wcw` branches of the jessetbh forks. If you change
  anything under `lib/`, the change belongs in the corresponding fork (PR against its
  `wcw` branch); the submodule pin here is then bumped to the pushed commit.
- A game-behavior patches pipeline (C cross-compiled to MIPS, like the sister
  project's `patches/`) is not stood up yet — do not hand-edit recompiler output
  (`RecompiledFuncs/` is generated and gitignored).
- Symbol identifications (naming a `func_XXXX`) need evidence — add it to
  `disasm/libultra.md` alongside the `RENAME` entry in `tools/gen_symbols.py`.
- Symbol metadata is tracked in-repo: `syms/dump.toml` is generated by
  `tools/gen_symbols.py` from the splat disassembly (`disasm/`) — regenerate it,
  don't hand-edit.
- Test on a real build before opening a PR: boot, play a match, open the config menu.

## Reporting bugs

Use the issue template. Always include your GPU + driver version, OS, the log output,
and your ROM's SHA1 (the only supported ROM is the US release, SHA1
`E1711A2511394B9357B5F1AC8CA5CC17BD674836`).
